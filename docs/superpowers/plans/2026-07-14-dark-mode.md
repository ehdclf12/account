# 다크모드 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 앱 전체를 라이트/다크 두 테마로 쓸 수 있게 하고, 홈/뒤로 버튼을 눈에 띄게 통일한다.

**Architecture:** 색을 CSS 변수 기반 시맨틱 토큰으로 바꾼다. `tailwind.config.js`의 색이 `rgb(var(--x) / <alpha-value>)`를 가리키고, `index.css`가 `:root`와 `[data-theme='dark']`에 값을 정의한다. 그러면 이미 토큰을 쓰는 387곳(`bg-card`·`text-ink`·`text-sub`·`bg-brand`)은 **코드 수정 없이** 다크에 대응된다. 손댈 곳은 하드코딩된 색 약 110곳뿐이다. `<html data-theme>`만 갈아끼우면 앱 전체가 한 번에 바뀌므로 React 리렌더가 필요 없다.

**Tech Stack:** React 19, TypeScript, Vite, TailwindCSS 3, Vitest (jsdom)

**Spec:** `docs/superpowers/specs/2026-07-14-dark-mode-design.md`

## Global Constraints

- **CSS 변수는 RGB 채널 값으로 정의한다** (`--brand: 49 130 246`, hex 아님). 코드에 이미 `bg-brand/10`, `border-sub/30`, `text-sub/50` 같은 투명도 조합이 있어서, hex로 넣으면 전부 깨진다. Tailwind 쪽은 `'rgb(var(--brand) / <alpha-value>)'`.
- OS(기기) 설정을 보지 않는다. 앱 안에서만 고른다. `prefers-color-scheme`를 쓰지 않는다.
- 기본 테마는 **라이트**. 저장은 `localStorage`의 `'theme'` 키, 기기별.
- UI 문구는 한국어.
- 새 npm 의존성을 추가하지 않는다.
- 테스트는 `src/lib/**` 순수 함수에만 붙인다. 이 프로젝트에 컴포넌트 테스트는 없으며 새로 만들지 않는다.
- **건드리지 않는 것:** `src/components/Donut.tsx`의 `CAT_COLORS` 7색, `src/lib/archive.ts`의 `ARCHIVE_COLORS` 5종. 둘 다 데이터를 구분하는 색이고 중간 명도라 다크에서도 읽힌다.
- `text-white` 31곳은 **그대로 둔다**. 파란 버튼 위 흰 글씨는 다크에서도 흰 글씨다.
- 각 태스크 끝에서 `npm test`와 `npm run build`가 모두 통과해야 한다.

- [ ] **Step 3: `theme.ts` 구현**

`src/lib/theme.ts` (신규):

```ts
export type Theme = 'light' | 'dark'

const KEY = 'theme'
// 상태바 색(PWA). 라이트=흰 배경, 다크=진회색 배경.
const THEME_COLOR: Record<Theme, string> = { light: '#FFFFFF', dark: '#17171C' }

export function loadTheme(): Theme {
  try {
    return localStorage.getItem(KEY) === 'dark' ? 'dark' : 'light'
  } catch {
    // 사파리 프라이빗 모드 등에서 localStorage 접근이 막힐 수 있다. 앱이 죽지 않게 한다.
    return 'light'
  }
}

export function saveTheme(t: Theme): void {
  try { localStorage.setItem(KEY, t) } catch { /* 저장 실패해도 이번 세션은 동작한다 */ }
}

export function applyTheme(t: Theme): void {
  document.documentElement.dataset.theme = t
  const meta = document.querySelector('meta[name="theme-color"]')
  if (meta) meta.setAttribute('content', THEME_COLOR[t])
}
```

`src/components/ThemeToggle.tsx` (신규):

```tsx
import { useState } from 'react'
import { applyTheme, loadTheme, saveTheme } from '@/lib/theme'
import type { Theme } from '@/lib/theme'

const OPTIONS: { value: Theme; label: string }[] = [
  { value: 'light', label: '☀︎ 라이트' },
  { value: 'dark', label: '☾ 다크' },
]

export default function ThemeToggle() {
  const [theme, setTheme] = useState<Theme>(loadTheme)

  function pick(t: Theme) {
    setTheme(t)
    saveTheme(t)
    applyTheme(t)
  }

  return (
    <div className="flex gap-1 bg-card rounded-full p-1">
      {OPTIONS.map((o) => (
        <button key={o.value} onClick={() => pick(o.value)}
          className={`rounded-full px-3 py-1 text-xs font-medium active:opacity-70
            ${theme === o.value ? 'bg-surface text-ink' : 'text-sub'}`}>
          {o.label}
        </button>
      ))}
    </div>
  )
}
```

`useState<Theme>(loadTheme)` — 함수를 그대로 넘겨 최초 렌더에서 한 번만 읽는다.
